=== 觀念題 ===
Q1：docker run -d 和 docker run -it 的使用時機有什麼不同？
A1：docker run -d（Detached）適合用於需要持續在背景執行的服務型容器，例如 nginx、資料庫等，容器啟動後不佔用終端機，可繼續執行其他指令。docker run -it（互動 + 終端機）適合用於需要直接與容器互動的情境，例如進入 Ubuntu 執行 bash shell 進行測試或除錯，會佔用目前的終端機。

Q2：-p 8080:80 的意思是什麼？如果把主機的 8080 改成 9090，應用程式的行為會有什麼不同？
A2：-p 8080:80 表示將主機的 8080 埠對應到容器內的 80 埠，外部瀏覽器透過 http://主機IP:8080 就能存取容器內跑在 80 埠的 nginx 服務。如果改成 -p 9090:80，容器內部 nginx 仍然監聽 80 埠不變，但外部必須改用 http://主機IP:9090 才能連線，原本的 8080 埠將無法存取服務。
