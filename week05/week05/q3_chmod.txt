Q1：請說明 chmod 755 代表什麼權限，擁有者、群組、其他人各能做什麼？
A1：
chmod	權限	擁有者	群組	其他人	常用情境
755	rwxr-xr-x	讀寫執行	讀和執行	讀和執行	Shell Script、可執行程式
Q2：請說明 chmod 644 代表什麼權限，跟 755 的差異是什麼？
A2：
chmod	權限	擁有者	群組	其他人	常用情境

Q3：為什麼 Shell Script 需要加上執行權限（chmod +x）才能用 ./ 執行？
    如果不加執行權限，還有什麼方式可以執行 Script？
A3：
為什麼需要 chmod +x 才能用 ./script.sh？

當你輸入：

./script.sh

系統會做這件事：

👉 嘗試把這個檔案「當作程式直接執行」

而要讓一個檔案能被「直接執行」，必須滿足兩個條件：

1️⃣ 檔案有「執行權限」（execute bit）

也就是：

chmod +x script.sh

否則系統會拒絕執行（Permission denied）

2️⃣ 系統知道怎麼執行它（Interpreter）

Shell Script 通常會在第一行寫：

#!/bin/bash

這叫 shebang，意思是：

👉「請用 /bin/bash 來執行這個檔案」

📌 總結：

條件	作用
+x 權限	允許檔案被當成程式執行
shebang	指定用哪個程式解讀








為什麼沒有 +x 就不能 ./script.sh？

因為：

👉 ./script.sh 是「直接執行檔案」

而不是「把檔案交給 shell 解讀」

系統會檢查：

你有沒有權限把這個檔案當程式跑？

沒有 → ❌ 拒絕

🔹 不加執行權限，還有哪些方式可以執行？

可以！而且很常用 👇

✅ 方法 1：用 shell 明確執行
bash script.sh

或：

sh script.sh

👉 這時候：

bash 是可執行程式
script.sh 只是「資料檔」

📌 重點：
👉 不需要 +x，因為你不是在執行 script，而是執行 bash

✅ 方法 2：用 source（或 .）
source script.sh

或：

. script.sh

👉 這會在「當前 shell」直接執行內容

📌 特點：

不需要 +x
會影響目前環境（例如 export）
✅ 方法 3：用其他 interpreter

如果是其他語言，例如：

python script.py
perl script.pl

👉 原理一樣：用 interpreter 讀檔

🔹 三種方式的差異
方法	需不需要 +x	是否新開 process	用途
./script.sh	✅ 需要	✅	像執行程式
bash script.sh	❌ 不需要	✅	最常見
source script.sh	❌ 不需要	❌	修改當前環境
🔹 一句話理解

👉 ./script.sh =「把檔案當程式執行」→ 需要 +x
👉 bash script.sh =「用 bash 讀這個檔案」→ 不需要 +x
