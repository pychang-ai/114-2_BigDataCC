=== 任務 6：觀念回答 ===
Q1：為什麼不要直接在 base 環境安裝專案套件？這樣做有什麼壞處？
A1：
不建議在 base 環境直接安裝專案套件，主要原因如下：

容易造成套件衝突
不同專案可能需要不同版本的套件，如果全部裝在 base，會互相影響，導致版本衝突或程式無法執行。
環境難以維護
base 環境會變得混亂，難以追蹤哪些套件是屬於哪個專案。
影響 Conda 本身運作
base 環境是 Conda 的核心環境，隨意修改可能會影響 conda 指令或其他環境管理功能。
不利於專案重現（reproducibility）
若沒有獨立環境，換電腦或與他人合作時，很難重建相同的執行環境。

👉 因此建議：每個專案建立獨立環境（conda env）
Q2：conda install 和 pip install 有什麼差異？什麼情況下會建議用 pip？
A2：
差異：

套件來源不同
conda install：來自 Conda repository（如 conda-forge）
pip install：來自 PyPI（Python 官方套件庫）
相依性管理
conda：會處理「跨語言」依賴（例如 C、C++ library）
pip：只處理 Python 套件依賴
環境穩定性
conda：較穩定，適合科學計算（numpy、pandas 等）
pip：較新、更新快，但可能較容易衝突
Q3：environment.yml 的用途是什麼？換電腦時要怎麼用它快速重建環境？
A3：
用途：
environment.yml 是用來記錄 Conda 環境的設定，包括：

環境名稱
Python 版本
已安裝套件與版本
套件來源（channels）

👉 可以用來「重現完全相同的開發環境」
