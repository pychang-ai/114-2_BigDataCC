=== 任務 1：探索 Linux 目錄結構 ===
執行以下指令：
ls /

（貼上結果）
bin                boot  etc   init  lib.usr-is-merged  lost+found  mnt  proc  run   sbin.usr-is-merged  srv  tmp  var
bin.usr-is-merged  dev   home  lib   lib64              media       opt  root  sbin  snap                sys  usr
請從上面的結果中，說明以下 5 個目錄各是做什麼的：
/home（家目錄）：
存放一般使用者的個人資料與設定檔。每個帳號通常會有一個獨立的子資料夾（如 /home/u306），使用者對此目錄擁有完整讀寫權限。
/etc（系統設定檔）：
存放系統所有的設定檔。例如帳號密碼檔 (/etc/passwd)、網路設定、啟動指令碼等，是管理系統最核心的目錄。
/var（變動資料）：
存放系統執行過程中會「動態變化」的檔案。常見的包括系統紀錄（Log）、郵件隊列、暫存快取或資料庫檔案。
/tmp（暫存檔）：
提供給系統或應用程式存放臨時檔案。通常在系統重啟後，這裡的內容會被自動清空。
/usr（軟體程式）：
存放系統安裝的軟體、指令、程式庫與說明文件。它的地位類似於 Windows 的 "Program Files"，大部分的使用者指令都放在 /usr/bin。

=== 任務 3：chown 與 chgrp 實作 ===
請依序執行以下指令並貼上每步的結果：

步驟 1：建立測試檔案並查看預設擁有者
touch ~/week06_test.txt
echo "Ownership Test" > ~/week06_test.txt
ls -l ~/week06_test.txt
（貼上結果，注意看擁有者和群組欄位）
-rw-r--r-- 1 u306 u306 15 Mar 31 16:29 /home/u306/week06_test.txt
步驟 2：建立測試用的使用者和群組
sudo useradd -m tempuser
sudo groupadd tempgroup
（貼上結果）

步驟 3：用 chown 改變擁有者
sudo chown tempuser ~/week06_test.txt
ls -l ~/week06_test.txt
（貼上結果，觀察擁有者欄位的變化）
-rw-r--r-- 1 tempuser u306 15 Mar 31 16:29 /home/u306/week06_test.txt
步驟 4：用 chgrp 改變群組
sudo chgrp tempgroup ~/week06_test.txt
ls -l ~/week06_test.txt
（貼上結果，觀察群組欄位的變化）
-rw-r--r-- 1 tempuser tempgroup 15 Mar 31 16:29 /home/u306/week06_test.txt
步驟 5：用 chown 一次改回自己
sudo chown $(whoami):$(whoami) ~/week06_test.txt
ls -l ~/week06_test.txt
（貼上結果，確認擁有者和群組都改回來了）
-rw-r--r-- 1 u306 u306 15 Mar 31 16:29 /home/u306/week06_test.txt
步驟 6：清理
rm ~/week06_test.txt
sudo userdel -r tempuser
sudo groupdel tempgroup
（貼上結果）
userdel: tempuser mail spool (/var/mail/tempuser) not found
