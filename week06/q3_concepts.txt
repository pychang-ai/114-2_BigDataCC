Q1：chown 和 chmod 有什麼不同？請分別說明它們的功能，並各舉一個使用情境。
A1：這兩個指令雖然都跟「權限」有關，但處理的層面完全不同：

chown (Change Owner)：

功能：用來修改檔案或目錄的「擁有者」或「所屬群組」。它決定了檔案歸誰管。

情境：助教把作業範例檔上傳到伺服器後，將檔案的擁有者改為學生，讓學生可以自由處置該檔案。

chmod (Change Mode)：

功能：用來修改檔案或目錄的「存取權限」（讀 r、寫 w、執行 x）。它決定了能對檔案做什麼。

情境：你寫好了一個 Python 腳本，使用 chmod +x script.py 讓它變成可以像程式一樣直接執行的檔案。

Q2：你和 3 位同學要合作做專題，需要在 Linux 上建立一個共用資料夾，
    讓組員都能讀寫檔案，但其他人不能存取。
    請寫出你會使用的指令（提示：需要建立群組、設定目錄的群組和權限）。
A2：假設專題名稱為 my_project，資料夾設在 /home/share，您可以執行以下步驟：

建立專題群組：
sudo groupadd project_team

將組員加入群組（假設組員帳號為 user1, user2...）：
sudo usermod -aG project_team user1 (以此類推)

建立資料夾並更改群組：
mkdir /home/share
sudo chgrp project_team /home/share

設定權限：
chmod 770 /home/share

7 (rwx)：擁有者有全權。

7 (rwx)：群組成員（組員）有全權，可讀寫。

0 (---)：其他人完全無法進入或查看。

Q3：Linux 判斷權限時，是按什麼順序檢查的？
    假設一個檔案的權限是 rwxr-x---，擁有者是 alice，群組是 project。
    請回答以下三個人分別能做什麼操作：
    (a) alice（擁有者）能做什麼？
    (b) bob（屬於 project 群組）能做什麼？
    (c) charlie（不屬於 project 群組）能做什麼？
A3：Linux 判斷權限時，會按 「擁有者 (User) > 群組 (Group) > 其他人 (Others)」 的順序由左至右檢查。一旦符合其中一個身分，系統就會直接套用該權限，不再往後看。

針對權限為 rwxr-x--- (750) 的檔案：

(a) alice (擁有者)：

權限：rwx

操作：可以讀取內容、修改/刪除檔案、也可以執行該檔案（如果是程式）。

(b) bob (屬於 project 群組)：

權限：r-x

操作：可以讀取內容、可以執行該檔案，但不能修改檔案內容（因為少了 w）。

(c) charlie (不屬於 project 群組)：

權限：---

操作：完全無法讀取或操作。如果這是一個目錄，他甚至連進去都沒辦法（沒有 x 就進不去）。
